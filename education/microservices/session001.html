Getting Started with Spring Boot

Before you Start

This session is intended for Java developers to get Started with Spring Boot. If you are familiar with Spring Boot, you may skip this session.

Getting Started with Spring Boot

  1. In your browser, visit http://start.spring.io.

  2. Fill out the Project metadata fields as follows:

    Group

    io.uslab

    Artifact

    hello-spring-boot

    Name

    hello-spring-boot

    Description

    Hello Spring Boot

    Package Name

    io.uslab.spring.hello

    Type

    Maven Project

    Packaging

    Jar

    Java Version

    1.8

    Language

    Java Spring Boot Version 2.0.5

  3. In the Project dependencies section, check the following:

    • Web

  4. Click the Generate Project button. Your browser will download a zip file. Unpack that zip file at $~/workspaces/lab001.

  5. Import the project’s pom.xml into your editor/IDE of choice.

  6. Add a @RestController annotation to the class io.uslab.spring.hello.HelloSpringBootApplication.

  7. Add the following request handler to the class io.uslab.spring.hello.HelloSpringBootApplication:

                                @RequestMapping("/")
                            public String hello() {
                                return "Hello World!";
                            }

Using Embedded Container

Spring Boot will embed Apache Tomcat by default.

  1. Change to the lab directory (or continue to build off of lab01):

    $ cd $workspaces/lab001/hello-spring-boot

  2. Build the application:

    $ mvn package

  3. Run the application:

    $ java -jar target/hello-spring-boot-0.0.1-SNAPSHOT.jar

    You should see the application start up an embedded Apache Tomcat server on port 8080:

                                        INFO 2465 --- [           main] s.b.c.e.t.TomcatEmbeddedServletContainer : Tomcat started on port(s): 8080 (http)
                                    INFO 2465 --- [           main] i.p.s.hello.HelloSpringBootApplication   : Started HelloSpringBootApplication in 3.023 seconds (JVM running for 3.432)

  4. Visit the application in the browser ( http://localhost:8080), and you should see the following:

    Hello World!

Using Jetty

Spring Boot also supports embedding a Jetty server.

  1. Open pom.xml and replace the following: 

                                <dependency>
                                <groupId>org.springframework.boot</groupId>
                                <artifactId>spring-boot-starter-web</artifactId>
                            </dependency>

    with:

                                <dependency>
                                <groupId>org.springframework.boot</groupId>
                                <artifactId>spring-boot-starter-web</artifactId>
                                <exclusions>
                                    <exclusion>
                                        <groupId>org.springframework.boot</groupId>
                                        <artifactId>spring-boot-starter-tomcat</artifactId>
                                    </exclusion>
                                </exclusions>
                            </dependency>
                            <dependency>
                                <groupId>org.springframework.boot</groupId>
                                <artifactId>spring-boot-starter-jetty</artifactId>
                            </dependency>
                            
                                        
                                            org.springframework.boot
                                            spring-boot-starter-test
                                            test

  2. Build the application:

    $ mvn package

  3. Run the application:

    $ java -jar target/hello-spring-boot-0.0.1-SNAPSHOT.jar

    You should see the application start up an embedded Jetty server on port 8080:

                                        INFO 3078 --- [           main] .s.b.c.e.j.JettyEmbeddedServletContainer : Jetty started on port(s) 8080 (http/1.1)
                                    INFO 3078 --- [           main] i.p.s.hello.HelloSpringBootApplication   : Started HelloSpringBootApplication in 3.671 seconds (JVM running for 4.079)

  4. Visit the application in the browser ( http://localhost:8080), and you should see the following:

    Hello World!

Using Externalized Configuration

Refactoring to externalize the configuration

  1. Change to the lab directory (or continue to build off of lab001 or lab002):

    $ cd $~/workspaces/lab001/hello-spring-boot

  2. Import the project’s pom.xml into your editor/IDE of choice (if you haven’t in a previous lab).

  3. Rename src/main/resources/application.properties to src/main/resources/application.yml. Into that file, paste the following:

    greeting: Hello

  4. To the class io.uslab.spring.hello.HelloSpringBootApplication, add a greeting field and inject its value:

                                            @Value("${greeting}")
                                        String greeting;

  5. Also io.uslab.spring.hello.HelloSpringBootApplication, change the return statement of hello() to the following:

    return String.format("%s World!", greeting);

  6. Build the application:

    $ mvn package

  7. Run the application:

    $ java -jar target/hello-spring-boot-0.0.1-SNAPSHOT.jar

  8. Visit the application in the browser ( http://localhost:8080), and verify that the output is still the following:

    Hello World!

  9. Stop the application.

  1. Run the application again, this time setting the GREETING environment variable:

    $ GREETING=Namaskar! java -jar target/hello-spring-boot-0.0.1-SNAPSHOT.jar

  2. Visit the application in the browser ( http://localhost:8080), and verify that the output has changed to the following:

    Namaskar! World!

  3. Stop the application.

Using Spring Profiles for Configuration

  1. Add a new spanish profile. Name it application-spanish.yml. Your finished configuration should reflect the following:

                                                    greeting: Hola

  2. Build the application:

    $ mvn package

  3. Run the application, this time setting the SPRING_PROFILES_ACTIVE environment variable:

    $ SPRING_PROFILES_ACTIVE=spanish java -jar target/hello-spring-boot-0.0.1-SNAPSHOT.jar

  4. Visit the application in the browser ( http://localhost:8080), and verify that the output has changed to the following:

    Hola World!

  5. Stop the application.

  1. Run the application, this time setting both the SPRING_PROFILES_ACTIVE and GREETING environment variables:

    $ SPRING_PROFILES_ACTIVE=spanish GREETING=Namaskar! java -jar target/hello-spring-boot-0.0.1-SNAPSHOT.jar

  2. Visit the application in the browser ( http://localhost:8080), and verify that the output has changed to the following:

    Namaskar! World!

  3. Visit http://docs.spring.io/spring-boot/docs/current/reference/html/boot-features-external-config.html to learn more about this outcome and the entire priority scheme for conflict resolution.
